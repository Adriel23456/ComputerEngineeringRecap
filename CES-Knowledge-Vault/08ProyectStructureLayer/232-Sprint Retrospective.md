## ğŸ“š Idea/Concept

The Sprint Retrospective is an opportunity for the Scrum Team to inspect itself and create a plan for improvements to be enacted during the next Sprint. It occurs after the Sprint Review and prior to the next Sprint Planning. This is a 3-hour time-boxed meeting for one-month Sprints (proportionally less for shorter Sprints). The Sprint Retrospective is the primary mechanism for continuous improvement in Scrum.

### Retrospective Techniques

| Technique | Description | Best For |
|-----------|-------------|----------|
| Start-Stop-Continue | Three columns of actions | Simple, regular use |
| 4 L's | Liked, Learned, Lacked, Longed For | Balanced reflection |
| Mad-Sad-Glad | Emotional check-in | Team feelings |
| Sailboat | Wind (helps), Anchors (slows), Rocks (risks) | Visual teams |
| Dot Voting | Prioritize items | Decision making |
| 5 Whys | Root cause analysis | Deep problems |
| Timeline | Events on Sprint timeline | Complex Sprints |

### Common Anti-patterns

- Same issues every Sprint (not addressing root causes)
- Too many action items (pick 1-3 max)
- No follow-through (actions forgotten)
- Blame game (not safe environment)
- Skipping the retro (continuous improvement stops)
